Security Operations Center is a dedicated facility operated by a specialised team 

This team aims to continuously monitor an organisation's network and resources and identify suspicious activity to prevent damage 
## Security Operations Center (SOC) Overview

The primary objective of a Security Operations Center (SOC) team is to maintain intact **[[Detection and Response]]** capabilities. They utilize centralized security solutions to continuously monitor the company's entire network and systems from a single location, aiming to detect and respond to security incidents.

## Core Focus: Detect and Respond

### 1. Detection
Continuous monitoring is required to identify various security events and threats:

* **Detect [[Vulnerability|Vulnerabilities]]:** Identifying weaknesses in device software (OS and programs) that an attacker could exploit.
    * *Note:* While fixing them isn't always the SOC's direct responsibility, unfixed vulnerabilities severely impact the company's overall security posture. (e.g., spotting unpatched Windows computers).
* **Detect Unauthorised Activity:** Spotting compromised accounts and unauthorized access attempts. 
    * *Indicators:* Anomalous clues such as unusual geographic login locations.
* **Detect Policy Violations:** Identifying breaches of the company's [[Security Policy]] (the rules designed to protect against threats and ensure compliance).
    * *Examples:* Downloading pirated media, transmitting confidential company files insecurely.
* **Detect Intrusions:** Identifying unauthorised access to systems and networks.
    * *Examples:* Successful exploitation of a web application, or a user infecting their machine by visiting a malicious site.

### 2. Response
* **Support [[Incident Response]]:** Once an incident is detected, the SOC assists the dedicated incident response team. 
    * *Key actions:* Minimising the incident's impact and performing a [[Root Cause Analysis]].

# The "People" Pillar of a SOC

Despite automation, human analysts are essential in a SOC. They filter out the overwhelming "noise" (false positives) generated by automated security tools to identify truly harmful activities and prevent wasted resources.

## [[SOC Team Hierarchy]]

* **SOC Analyst (Level 1):** First responders. Perform basic [[Alert Triage]] to verify if a detection is harmful and escalate appropriately.
* **SOC Analyst (Level 2):** Deep investigators. Handle complex detections by correlating data from multiple sources for thorough analysis.
* **SOC Analyst (Level 3):** Advanced responders. Proactively hunt for [[Threat Indicators]] and manage critical [[Incident Response]] (containment, eradication, recovery).
* **[[Security Engineer]]:** Deploy, configure, and maintain the security solutions used by the analysts.
* **[[Detection Engineer]]:** Create the rules and logic used by security tools to detect threats (a role sometimes filled by L2/L3 analysts).
* **SOC Manager:** Oversees processes, supports the team, and reports the security posture to the [[CISO]].

> *Note: The exact structure and number of roles vary based on organization size.*
> 
# The "Process" Pillar of a SOC

Processes are the standardized workflows that the SOC team follows to handle security alerts effectively and efficiently. 

## 1. [[Alert Triage]]
The foundational first step for handling any alert. Triage determines the severity and priority of a detection by answering the **5 Ws**:

| The 5 Ws | Description / Example (Malware on Host) |
| :--- | :--- |
| **What?** | What exactly happened? *(e.g., A malicious file was detected inside the network.)* |
| **When?** | The exact timestamp. *(e.g., Detected at 13:20 on June 5, 2024.)* |
| **Where?** | The specific location/device. *(e.g., In the directory of host "GEORGE PC".)* |
| **Who?** | The user or entity involved. *(e.g., The file was detected for the user 'George'.)* |
| **Why?** | The motive or root cause. *(e.g., User downloaded it from a pirated software site to get it for free.)* |

## 2. Reporting
Harmful alerts must be properly escalated to higher-tier analysts to ensure a timely resolution. 
* **Method:** Logged and escalated as assigned tickets.
* **Requirements:** A complete report must include the 5 Ws, a thorough analysis, and screenshots used as evidence of the malicious activity.

## 3. [[Incident Response]] & Digital Forensics
Triggered when detections point to critical, highly malicious activities.
* **Incident Response (IR):** Specialized, higher-level teams step in to contain, eradicate, and recover from the threat.
* **Forensics:** The deep-dive analysis of system or network artifacts to determine the exact [[Root Cause Analysis|root cause]] of the security incident.

# The "Technology" Pillar of a SOC

Security solutions centralize network data and automate [[Detection and Response]], drastically reducing the manual effort required to monitor individual devices.

## Core Security Solutions

* **[[SIEM]] (Security Information & Event Management):** Collects and correlates logs from across the network to trigger alerts based on rules, User Behavior Analytics, and ML. *Note: Primarily provides **Detection** capabilities only.*
* **[[EDR]] (Endpoint Detection & Response):** Delivers deep, real-time visibility into individual devices (endpoints) with both detection *and* automated **Response** capabilities.
* **[[Firewall]]:** A network barrier that monitors and filters incoming/outgoing traffic to block threats before they reach the internal network.

## Additional Technologies
Depending on an organization's resources and threat surface, SOCs may also deploy: [[Antivirus]], [[EPP]], [[IDS/IPS]], [[XDR]], and [[SOAR]].
