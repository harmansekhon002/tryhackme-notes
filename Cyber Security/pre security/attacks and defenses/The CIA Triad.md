 In cyber security, being secure means ensuring specific conditions for the digital data. These conditions are the key aspects around which the security revolves.

- **Confidentiality**
- **Integrity**
- **Availability**
or ***CIA***
## confidentiality
- the fact that sensitive data can only be accessed by authorised individuals only 
- if anyone can access private data then the data might be at risk 
- this can result in financial loss privacy violations or legal consequences
## integrity 
- this refers to the situation where only an authorised individual can make changes to personal or sensitive business data
- this can result in severe financial loss or personal values vulnerability 
- lets suppose a situation where we are making a transaction and someone intercepts the transaction and changes the receiving info
## availability
- it ensures that the data is available to authorised users at the correct time and when needed 
- even a short amount of down period may lead to a huge financial or other loss
- cyber attackers use this to their advantage where they send a large number of requests to a website making it unable to handle them all making the website fail or the server down
- even this can cause a lot of problems for the company 
- to prevent this websites are made so that the traffic is managed and also the api rates are limited
## security mindset
 
CIA is not just a set of definitions, it's a security mindset of cyber security professionals. When a security incident occurs, it is often explained in terms of what was affected. Security professionals generally by asking questions like:

Was sensitive data exposed to unauthorized individuals?

Was data being modified without permission?

Were systems or services unavailable to users when they needed?
